# claude-history

## Overview
- **Repo Name:** `claude-history`
- **Category/Family:** CLI / Terminal UI (TUI) / Session Manager / History Searcher
- **Purpose:** A companion CLI tool for Claude Code. It provides a fuzzy-searchable, ledger-style terminal interface to browse, read, search, export, resume, and fork recent conversations recorded in Claude's local project history.
- **Provider Coverage:** Claude Code (Anthropic's local CLI agent).

## Ingestion, Storage, and Update Strategy
- **Ingestion:** Reads directly from the local filesystem where Claude Code stores its logs (`~/.claude/projects/`). The directory names are lossily encoded paths generated by Claude.
- **Storage Strategy:** Zero persistent database. It operates fully ephemerally by globally traversing the `~/.claude` directory and parsing `.jsonl` session files on the fly. 
- **Update Mechanism:** There is no background watcher daemon. Updates occur via a full reload upon startup. To keep the UI responsive, it employs a streaming background loader using `rayon` to parse JSONL files in parallel and stream batches of `Conversation` objects into the Ratatui-based TUI via message passing (`std::sync::mpsc`).
- **Indexing:** Searches are performed in-memory. It maintains a `full_text` index for each conversation which truncates massive tool outputs to a maximum of 16KB to bound memory/CPU constraints.

## Transcript Structure Model
The internal data model strictly aligns with Claude Code's streaming log entries. 
The core entity is a `Conversation`, generated from parsing lines of a JSONL file:

```rust
pub enum LogEntry {
    Summary { summary: String },
    User {
        message: UserMessage,
        timestamp: Option<String>,
        cwd: Option<String>,
        parent_tool_use_id: Option<String>,
    },
    Assistant {
        message: AssistantMessage,
        timestamp: Option<String>,
        parent_tool_use_id: Option<String>,
    },
    System { subtype: String, duration_ms: Option<u64>, ... },
    CustomTitle { custom_title: String },
    Progress { data: serde_json::Value, ... }
}
```

- Content is broken down into `ContentBlock` enums (`Text`, `ToolUse`, `ToolResult`, `Thinking`, `Image`), allowing the UI to easily show/hide thinking blocks and truncate large tool outputs.

## UX Shape
- **Built with:** Rust, `ratatui` (TUI), and `crossterm`.
- **List Mode:** A fuzzy-finder list view (fzf style). Allows filtering by global scope vs. current workspace scope. Shows timestamp, project path, duration, token usage, model, and a conversation text preview.
- **Viewer Mode:** Triggered via `Enter`. Displays the conversation in a heavily formatted, scrollable ledger reading experience. 
  - Allows cycling tool call visibility (`off` / `truncated` / `full`).
  - Allows toggling Claude's "Thinking" blocks.
  - Supports `/` for in-conversation search.
  - Contains actions to "Resume" or "Fork and Resume" which delegates execution directly back to `claude --resume <uuid>`.

## Strengths
- **Incredibly Fast and Ephemeral:** Because it doesn't maintain a secondary stateful DB, it doesn't suffer from sync/drift issues. Heavy parallelization makes it load rapidly anyway.
- **Native Context & Forking:** Deeply understands Claude's concepts of sessions, subagents, and tools, allowing it to provide 1:1 actionable commands like forking or resuming directly in Claude Code.
- **Granular Viewing Options:** The ability to dynamically fold/unfold thinking tokens and large CLI tool outputs makes traversing raw transcripts highly readable compared to pure JSON.

## Weaknesses
- **Tight Coupling:** The architecture is entirely coupled to Claude Code's undocumented and internal `.claude/projects/` filesystem structure and JSONL schemas. If Anthropic changes this schema, `claude-history` will break.
- **Scalability Ceiling:** Relying on on-the-fly parallel parsing of all `.jsonl` files works well for hundreds of files but will linearly degrade in CPU/memory usage as the user accumulates thousands of heavy LLM sessions over time.
- **Single Provider:** Designed exclusively for Claude Code, possessing no generic provider-agnostic interface.

## Key Code Snippets

### Background Parallel Loader Streaming
*Demonstrates how it avoids UI blocks by parallelizing JSONL parsing and streaming batches back to the TUI.*
```rust
// src/history/loader.rs
pub fn load_all_conversations_streaming(...) -> Receiver<LoaderMessage> {
    let (tx, rx) = mpsc::channel();
    thread::spawn(move || {
        let projects = list_projects(&root).unwrap();
        projects.par_iter().for_each(|project| {
            let project_dir = root.join(&project.name);
            if let Ok(mut convs) = load_conversations(&project_dir, show_last, debug_level) {
                let _ = tx.send(LoaderMessage::Batch(convs));
            }
        });
        let _ = tx.send(LoaderMessage::Done);
    });
    rx
}
```

### Path Decoding Heuristic
*Demonstrates a fragility resulting from tight coupling; reverse-engineering Claude's directory encoding logic.*
```rust
// src/history/path.rs
/// Decode a project directory name back to a path (simple heuristic fallback).
/// Claude's encoding replaces all non-alphanumeric characters (except `-`) with `-`.
/// This means `/`, `_`, and `.` all become `-`, making the encoding lossy.
pub fn decode_project_dir_name_to_path(encoded: &str) -> PathBuf {
    PathBuf::from(decode_with_double_dash_as(encoded, "__"))
}
```
